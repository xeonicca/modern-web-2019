JavaScript 轉移至 TypeScript 的快速通道

陳孝勇 (Luke)

IBM

TypeScript 是一個由 MicroSoft 開發的 JavaScript 的嚴格超集語言，它幫 JavaScript 增加了的型別定義，讓 JavaScript 變得更嚴謹，同時程式可讀性和可維護性也同時提高了。但關於把專案從純 JavaScript 轉移至 TypeScript，TypeScript 官網建議大家將專案裡的 .js 檔案改成 .ts，然後 it’s not unexpected to get error messages after conversion. The important thing is to actually go one by one through these and decide how to deal with the errors. 翻譯就是，＂預期在轉移至 TypeScript 後會有錯誤訊息，重要的是你要一個個去處理這些錯誤＂。這樣一句簡單的話沒有告訴你的是，如果你的專案轉移至 TypeScript 後有九千多個錯誤，你該怎麼做？如果一天一個人可以處理 50 個錯誤，我要花近 200 個人/天來完成，同時還沒考慮程式會繼續開發所產生的其它錯誤...

這個議程會跟聽眾分享我們專案的實際經驗，分享我一個人如何在一個月內，將九千多個 type error 降到不到一千個，除了研讀 TypeScritp 文件之外，主要是詳細分析所有的錯誤，做分類，再去了解每類型錯誤可能的處理方法，接著找出可能解決的共同特徵，並懂得利用撰寫 Script 來協助處理，當然還有許多細節和觀念要注意，現場也會展示從九千多個 Type error 到一千個的過程。

我們不是微軟，不是開發 TypeScript 的公司，大家都知道 TypeScript 的好處，不用多提，但我們更知道，當專案想嘗試採用 TypeScript 時，第一個障礙就是上百上千個 Type errors 在眼前，讓我們很容易放棄，接著說服自己，反正一直以來用純 js 寫都沒問題，不必要轉移至 TypeScript 也沒關係，但其實你已經錯過很多 TypeScript 帶來的好處了。希望分享我們專案的實際經驗後，讓大家可以更輕易的跨進 TypeScript 的世界。
